RankUp Education Mobile — project run guide

The RankUp Education Flutter app lives at Mobile/ in this monorepo (package name rankup_education). android/ and ios/ wrappers are already present — do not run flutter create . unless regenerating native projects on purpose.

1. Start the Web API

# From repo root — bind all interfaces so the Android emulator can reach the host
dotnet run --project WebApi/src/RankUpEducation.Api --launch-profile http
# applicationUrl should be http://0.0.0.0:5255 (not localhost-only)

2. Start the Flutter app

cd "D:\Projects\RankUp Education\Mobile"
flutter pub get
# Start emulator in another terminal first: emulator @Pixel_7_API_36
flutter devices
flutter run

3. API base URL by device

TargetDefault / recommended URLNotes
Android emulator (default)http://10.0.2.2:5255/api10.0.2.2 is the emulator alias for the host machine. localhost inside the emulator is the emulator itself.
iOS Simulator / desktop / webhttp://localhost:5255/apiDefault when not Android (see lib/app/api_base_url.dart).
Physical Android devicehttp://<YOUR_PC_LAN_IP>:5255/apiUse --dart-define=API_BASE_URL=... and ensure PC firewall allows port 5255. 10.0.2.2 will not work on a real phone.

Resolution lives in Mobile/lib/app/api_base_url.dart and Mobile/lib/app/environment.dart.

4. Dart defines

# Default: real API, development env, Android emulator base URL
flutter run

# Explicit API URL
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:5255/api

# Offline mock auth (demo accounts only)
flutter run --dart-define=USE_MOCKS=true

# Environment name: development | test | staging | production
flutter run --dart-define=APP_ENV=development

With default API mode, login calls the real Web API (POST /api/auth/login). Use a real account from PostgreSQL.

6. Cleartext HTTP for local API

  • Main manifest references res/xml/network_security_config.xml allowing cleartext to 10.0.2.2, localhost, and 127.0.0.1.
  • debug/ and profile/ manifests set android:usesCleartextTraffic=true for local HTTP during development.
  • WebApi Development profile must listen on 0.0.0.0:5255 so the emulator can connect.

SymptomWhat to check
Connection refused / timeouts on loginAPI not running, or bound to localhost only. Use launch-profile http with 0.0.0.0:5255. Confirm with flutter logs that base URL is http://10.0.2.2:5255/api on emulator.

iOS Development Requirement

Flutter iOS code can be written on Windows, but the iOS simulator and iOS build chain require macOS. For iOS testing and release builds, prepare a Mac with:

  • macOS compatible with the current Xcode version.
  • Xcode installed from the Mac App Store or Apple Developer downloads.
  • Xcode command-line tools configured with xcode-select.
  • iOS Simulator runtime installed through Xcode.
  • CocoaPods installed for Flutter plugins that need native iOS dependencies.
  • Flutter SDK installed and checked with flutter doctor -v.

Xcode cannot replace the Android SDK for Android builds. It is required for iOS, while Android still needs the Android SDK, platform tools, build tools, platform package, and emulator or physical Android device.

When to Install a Larger Android System Image

The current Android emulator uses a smaller default x86_64 image for an optimized setup. Install a Google APIs or Google Play system image only if the app requires emulator-side Google Play Services, Google Maps, Play Billing, Firebase auth flows that depend on Google services, or Play Store testing. RankUp Mobile declares Firebase packages; local API login and Approvals work on the default image without Play Store.
